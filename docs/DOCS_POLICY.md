# Docs Policy

## 目的
- 実装時の一次情報を `docs/` に集約し、再開時の探索コストを下げる。

## 参照ルール（index-first）
1. 作業開始時に `docs/INDEX.md` を読む。
2. `Must-read` のみ必ず読む。
3. それ以外は作業に必要な文書だけ読む。
4. `docs/WORKLOG.md` は再開・デバッグ時のみ読む。

## 更新ルール
- 仕様変更時は `docs/REQS.md` を先に更新する。
- 重要リファクタ/複雑機能は ExecPlan を作る。
- 文書の追加・削除・用途変更時は `docs/INDEX.md` を更新する。
- チェックポイント到達時と入力待ち停止前に `docs/WORKLOG.md` へ追記する。
- push / release 前の公開衛生ルールを変える時は `docs/PUBLISHING.md` も更新する。

## 記述ルール
- 受け入れ条件は「人間が観測可能」な形で書く。
- 可能な限りコマンドと期待結果をセットで記載する。
- 断定できない事項は前提・仮定として明記する。
